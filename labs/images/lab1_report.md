- Для сервисного аккаунта роль была изменена с `Storage Admin` на `Compute Viewer`.

## Аналитика и выводы
- Роли IAM напрямую определяют доступность операций в облаке.
- Роль `Storage Admin` предоставляет полномочия для чтения/записи объектов в бакете.
- Роль `Compute Viewer` предназначена только для просмотра ресурсов Compute Engine и не дает прав на Cloud Storage.
- Минимальная Spot-VM (`e2-micro`) подходит для учебных задач и коротких операций через CLI, снижая стоимость.
- Принцип минимально необходимых привилегий подтвержден на практике: после урезания прав чувствительные операции становятся недоступны.
